🚀 Project Overview

Retro Engine Studio is an experimental web-native game creation suite. It combines a 2D grid-based editor with a 3D Three.js renderer to create "2.5D" FPS experiences similar to Doom or Wolfenstein 3D.

🏗 Level Structure (JSON)

Levels are stored as a single gameData object. This object contains the palette of assets and the 14x14 grid maps.

{
  "title": "Level Name",
  "assets": { "key": "base64_data" },
  "palette": {
    "walls": [{ "id": 1, "name": "Brick", "tex": "key" }],
    "enemies": [{ "id": 10, "hp": 100, "speed": 0.05 }]
  },
  "map": {
    "floors": [[101, ...]],
    "entities": [[99, ...]]
  }
}

Core ID Constants

Certain IDs in the entities map are reserved for core engine logic:

  • 0: Empty Space
  • 99: Player Spawn Point (One per map)
  • 98: Goal / Level Exit

🎨 Asset Creation

The editor includes an embedded PixUrl Utility. When you create an asset, you can draw it directly in the tool. The engine captures the image-processed event and automatically binds the resulting data to your asset.

🔗 Sharing & Persistence

Currently, the engine uses Base64 URL encoding for sharing. When you "Save" or "Share", the entire level state is stringified and appended to the URL hash. There is no backend database; the URL is the level.

🧪 Local Development

To run the test suite locally:

npm install
npm test          # Run Jest Unit Tests
npm run test:e2e  # Run Playwright E2E Tests
