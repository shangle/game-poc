The Future: Generating Games with LLMs and Image Templates

Our journey began with encoding images into URLs. Now, we are looking at the inverse: translating images into fully playable, interactive 3D worlds using Large Language Models (LLMs).

The ultimate goal of the Retro Engine architecture is to empower an LLM to create a game you can play instantly.

The Vision: Prompt to Play

Imagine providing an AI with a simple prompt: "A spooky dungeon with a central treasure room, surrounded by goblin guards."

Rather than the AI trying to write thousands of lines of complex Three.js code, our engine allows the AI to simply generate a JSON state map. Because our engine handles all the rendering, physics, and collision logic, the AI only needs to know where things go, not how to draw them.

Image Templates as Blueprints

Taking this a step further, an LLM with vision capabilities can analyze an Image Template. A user could draw a simple 14x14 grid in MS Paint or via a web tool:

  • Black pixels = Walls
  • Blue pixel = Player Spawn
  • Red pixels = Enemies
  • Green pixel = Exit

The AI acts as the translator. It looks at the image template, maps the precise positions for players, bad guys, floors, and walls, and trivially compiles this into our engine's format. The result is imported seamlessly, and the user can immediately begin playing.

Local AI, Global Capabilities

This engine represents a shift in software development. It is one in hopefully a series of small, local applications that leverage the immense power of AI. The software itself is lightweight, but when paired with an LLM's ability to generate data, it becomes infinitely expansive.

The barrier to game creation is no longer code. It's simply imagination.
